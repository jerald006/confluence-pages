XeniApp : Authentication

Created by Gerardo Nizetich on Mar 26, 2024
  • Summary
  • Example payload

Summary

The provider required to create a session to concatenate on the next api calls.

Example payload

/CreateSessionRQ
curl --location --request POST 'https://restapidemo.myfarebox.com/api/CreateSession' \
--header 'Content-Type: application/json' \
--header 'Authorization: {{apiKey}}' \
--data-raw '{
  "AccountNumber": "{{accountNumber}}",
  "Password": "{{password}}",
  "UserName": "{{userName}}"
}'
/CreateSessionRS
{
    "Success": true,
    "Data": {
        "SessionId": "20368AB0-5B31-4DF3-BFB9-327D314A191E-5260"
    }
}

The SessionId must be concatenated to the next api calls in the header of the request as Authorization: Bearer{{sessionId}}
