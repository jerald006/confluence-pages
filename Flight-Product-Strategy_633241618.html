Summary

The current Flight product is not attractive for customer, it brings complains and some customer abandon the use of it and even the Xeni App.

The proposal is to change the layout and improve the functionalities and content according with the industry and the competitors. Offer more content and more functionalities based on an standadar path.

Shopping => offers-management

Booking => order-management

This proposal includes the enhancements on:

  • Xeni FE:

    • Flights presentation

    • Booking Flow

    • Check-out.

    • Payment.

    • Use metrics

  • Xeni BE

    • Standardize the Xeni APP according to IATA Standards. (AIDM)

    • Cache data and Cache Rotation by Aggregator

    • Market Tables logic. (Preferred Vendors by Market)

    • Audit logs

    • Monitors

    • Notifications

  • Xeni APP administration

    • Flight Product configuration Dashboard

    • Content Management by Market/Vendors.

  • Data Analysis

    • Create the data structure to analyze consumer behavior

    • Get the metrics of use to identify possible improvements

    • Metrics to identify tendencies and peak of consumption by Market/Vendors.

Product description

Current:

The current product concentrate all the content from Thomalex and that is a pain point to make it scalable. We are in dependant of Thomalex´s capacity to delivery and integrate new aggregators content as well as functionalities. (Ex: post booking), also, we have an strong dependency of content management and no control over issues.

Proposal

Re-lunch the Flight product in Xeni App to give user a better experience and allow us to control the content and quality for the product.

The strategy includes:

  • Xeni Flights API: Create the internal API for Flights based on the standards of the industry and align with the AIDM concepts.

  • Decouple aggregators: Direct connection with the aggregators instead consume through Thomalex to have more control and scalability of the functionalities and offers. (Consolid and Travel Fusion), in different data models (Include NDC Connections)

  • Metrics that Matters: Create the metrics funnel to analyze the behavior of consumers and providers to set smart offer by Market/consumer/provider.

  • Share: Analysis of consumption by Market and set preferred carriers by Market with the objective of concentrate segments share.

  • Direct Integrations: Based on the data analysis of Share, investigate the options to direct integration with Airlines (NDC) and GDS providers to replace aggregators.

Functional Objectives

  • Xeni Flights API : Create a standard set of endpoint to cover the list of functionalities to satisfy our flight flow model.

    • Offer-management: (It is related to the flights itineraries and prices)

      • Flights shopping (Flight itinerary with Prices)

      • Itinerary Upsell (Alternative fares for given itinerary - Branded Fares)

      • Offer Prices (Ancillaries and Extras)

    • Order-management: (Related to Bookings)

      • Create Order (Create a booking PNR)

      • Seat map (Available seat for selected cabin/flights)

      • Change Order (Modify the PNR)

      • Cancel Order (Cancel PNR)

      • Purchase Order (Ticketing)

      • Retrieve Order (Retrieve reservation details).

      • Notification Order (Messages related to a PNR)

  • Application Backend

    • Identify the vendors that offer the acconding functionalities:

      • Shopping Service

      • ConfirmShopOption

      • Upsell

      • Sea tmap

      • Ancillaries

      • Hold Bookings

      • Change booking

      • Retrieve booking

      • Cancel Booking

      • Post-booking (Exchange)

        • shopping exchange

        • exchange order (ticket)

    • Create a decorator to avoid duplicity of offers

  • Application Front-end

    • Enhance flight widget with “advanced search” options.

    • Add the basket summary

    • Add Seat map functionality

    • Add the Ancillaries offers

    • Handle the split payment when the providers allow it.

  • Metrics to measure (We need to introduce the metrics and data analysis to identify the behavior of consumptions)

    • Top Destinations

    • Top Carriers preferred

    • Top Sellers

    • Top Vendors content provided

    • Top Vendors performance

    • Market-share by carrier

  • NDC:

    • Analyze the Data modelling to include NDC as integration model

  • Activity Logs

    • Create a log for shopping consumption

    • Create a separate logs for transactional consumption

    • Identify the consumption by customer, per market, per vendor.

Integration content

It is required to set 2 different integration strategy to achieve the final objective.

Multi-vendors

  • Middleware to handle vendors direct integration

  • Decorator: Filter options and keep the cheapest one

  • Dynamic Cache: Create a cache logic for the fared option based on the parameter of search

  • Cache Rotation: Create the logic to update the cache data.

  • Standardize the payload with XENI API Flights.

Model

Airlines Direct integration

  • Adopt the NDC model to direct integrate the “Preferred Airlines” (The most consumed airlines through Xeni APP)

  • Based on analytic data, create an strategy team to deal special sale conditions with the Vendors (Aggregators and airlines by market)

Development Strategy

The development strategy proposed is the MVP model starting with direct integrations with new aggregators and decouple those vendors integrated through Thomalex like consolid and Travel Fusion.
This MVP should be incremental, we need to determine which will be the best pilot, create the backend and administration side, create the metrics to measure the activity and success and possible deviations, create the proper logs.

MVP1:

  • Enhance the XENI API Flights.

  • Create the Market Logic (Market Tables and Preferred provider)

  • Create the Super Admin Dashboard to control the content.

    • Create the business Rules

  • Create the direct integration with CONSOLID.

  • Create a decorator to void duplicates.

  • Decouple Consolid from Thomalex.

  • Select the Pilot beta customers

  • Create the Data Funnel to compare Thomalex vs Consolid.

  • Measure the success

MVP2:

  • Create the direct integration with TRAVELFUSION.

  • Adjust the decorator to void duplicates.

  • Decouple travel fusion from Thomalex.

  • Select the Pilot beta customers

  • Create the Data Funnel to compare Thomalex vs Travel Fusion and Consolid.

  • Measure the success

  • Incorporate the NDC Content of Travel Fusion.

MPV3:

  • Increase content and functionality based on Integration prospects like Mystifly.

  • Analyze the best provider that return the best option in GDSs.

Release Strategy

I propose to make a canary release:

  • Select an Agency and enable the new Flight path.

  • Work with the Beta Agency to catch the feedback and possible deviations.

  • Monitor the activity

  • Measure the shopping process vs the bookings confirmed and the ticketing PNRs.

  • Correct any deviation

  • If success: Add more beta agencies.

  • Control de activity and make the decision to go/ no go for all the users.
