XeniApp : Auditing requests and responses to Vendors

Created by Rajesh, last modified on Dec 07, 2021

This document provides outline on how to audit requests sent to vendors and responses received from vendors.

What are things which we want to audit, this list is evolving and keeps updating

  1. requests sent to vendor

  2. responses received from vendor

  3. network failures after calls being sent to vendor

  4. application crashes after calls being sent to vendor

We have identified the following products which would help us to audit

  1. AWS firehose

  2. Segment

  3. Amplitude

  4. AWS SNS

From implementation perspective, we have two options

  1. Using a express middleware

  2. using axios interceptors

Using express middleware

  1. In this approach we create a custom Action

  2. the controller when making an api call it triggers the action

Using axios interceptor

  1. create axios wrapper which extends axios

  2. use axios request and response interceptors to capture requests and reponses

  3. write to service which is finalized from the top list
