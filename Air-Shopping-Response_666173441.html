XeniApp : Air Shopping Response

Created by Gerardo Nizetich on Jun 05, 2024

The response from the vendor is a collection of dictionaries associated with ReferenceIDs, the structure is quite similar to the NDC or OTA standards of the industry except for the Origin Destination components that are internally defined by LegIndicator

The LegIndicator is used to order the sequence of the segments, it refers to directionality

LegIndicator = 0, represents the first component of the trip and sequentially refers to the next components.

Parse sequence

The logic to define each offer to display, always start from the Obj: PricedItineraries

Each price itinerary will refer to a group of combined flight segments (JOURNEY). It represent a price instance of the itinerary.

From this point, it is required to:

1 - Identify the Itinerary:

2 - Identify the different prices offered for the itinerary

3 - Associate each fare family priced for the same itinerary in the same cabin

4 - Associate the Penalties policy and Fare price details.

Once each itinerary (Journey) has identified all the prices instances, it should be transformed in the XENI_API /offer-management/airshopping[ResponseBody]

The examples of different type of shopping response are not included here as the request for the size and weight of each.
