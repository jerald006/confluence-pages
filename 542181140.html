XeniApp : Events Admin/Client

Created by User ec155, last modified on Nov 07, 2022

Upon login:

User goes to Dashboard → this shows the user:

Total Events

Open Events

Accounts Manages the Administration Accounts

Admin Accounts can be created based through: Users – https://eventsadmin.xeni.com/admin/accounts/user

Sessions (logins) can be seen via User Tokens

Order of operations: (metadata can be added for internal use only)

  1. Create an event:

    1. Requires a name

    2. Seed amount for wallet - should be 1.1

    3. url → defined URL

  2. Create a raffle:

    1. Name

    2. Entry Price

    3. Hours and Minutes

    4. Event association

  3. Create a or many prizes:

    1. Name

    2. Description

    3. Raffle association

    4. image (to be added)

  4. Start Raffle (back on Raffle Model)

User Settings:

Change associated e-mail or manage password.

Client side:

As discussed user should enter flow through browser → this is the case for iPhone and Android (Safari and Chrome), entering through Slack creates issues.

User should enter details and download JSON file with credentials or screenshot.

Next user should Buy Raffle entry (currently they get a successful entry from the TX)

Once the timer hits 0 – the winners will pop-up.
