Xeni WordPress downtime due to a database connection error

Date: 8 Dec 2022

Authors: User 7c674 User c94cc

Status: Fixed

Summary: Xeni WordPress website was down due DB connection error. During this period xeni.com was not available but xeni booking engine was working fine.

Impact: Potential revenue loss.

Root Causes:

On Dec 8, 2022, 5:56:15 PM UTC / Dec 8, 2022, 12:56:15 PM EST, the WordPress database “options” table crashed and the table auto recovery has been failed.

The WordPress website could not connect with a database hosted on the same AWS instance. Due to this, we are getting “Error establishing a database connection” error. After restarting the server, the issue got fixed.

Resolution:

The following Action will take to resolve this issue and this will make xeni.com as 99.9% uptime.

  • Planned to move MySQL database to AWS RDS instance.

  • Planned to Push the code to GitHub

  • Planned to implement the CI/CD process for WordPress.

  • Planned to Create the AMI with required deps.

  • Planned to implement the Load Balancer and Auto Scaling feature.

  • Planned move static assets to S3 bucket and make it available on CDN.

Timeline:

  • Sachin Narode reported this issue at 2:19 PM EST.

  • User 7c674 restarted the server and the outage was mitigated.
