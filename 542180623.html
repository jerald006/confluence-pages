Process for the Developers

  1. Whenever a task/bug is assigned, as soon as you start working, Change the status to - Selected for Development

  2. Once the development is done, merge it to Staging and inform the Reporter to test the issue (Ofcourse test it on your local before merging it to Dev). Change the status to - Ready To Be Tested

  3. Write any comments on exceptions made in the development and who had approved it

  4. Mention the link of the PR in the comments section

  5. A screenshot of the task is to be attached in the PR if possible

Process for the Testers

  1. Once Testing is started - Change the status to - Testing in Progress

  2. If any issues, mention them in the comments and inform the developer

  3. Attach a list of tests performed

  4. If the issue is resolved, Change the status to - Ready for Production
