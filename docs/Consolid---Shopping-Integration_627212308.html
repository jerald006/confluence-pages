Overview

Get the Consolid content and display the Itinerary Offer for a certain Origin Destination detailed.

Proposal Flow:

Proposal

Based on direct integration and have the right metrics about the consumption behavior by Market, it is possible to create a Cache of the Top 10 Destinations with a updating logic and use the method /confirmFlightAvailability to be sure that the option offered from Cache is available before display to the user.

This need to be deeply analyze for fight implementation.

Shopping response structure

Each fare option, offer different segments combinations to create the itinerary (journey) associated to a FareId.

This FareId is used to reconfirms the availability of the offer before presenting to the consumer.

The performance of the endpoint is between 10s to 35s depending the route and the concurrent carriers offers.
Each fareid is stored in Consolid side for 40 Minutes, which give the option to build a cache.

/offers-management/airShopping - data

  • Contains the result of the search, the structure of the data is based on a shoppingID that will be the referenceKey for any future uses of the data.

  • shoppingId: This property contains a UUID that will reference the specific set of data for a period of (N) time. The shoppingID is an internal of XENI that reference the connection with 1 or more than one vendor.

  • originDestinationOptions: Contain the list of flights by directionality key.
