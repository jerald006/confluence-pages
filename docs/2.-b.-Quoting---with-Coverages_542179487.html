XeniApp : 2. b. Quoting - with Coverages

Created by User d8ec2, last modified on Jun 09, 2021

When quoting you have the option to supply coverages within the coverages array node.

When quoting in this manner, the optionalCoveragesQuoteAmount only apply to the optional coverages you passed in and, only optional coverages that are passed in are returned. Included coverages are always returned.

Request

 {
    "policy" : {
        "productId" : "EC",
        "isQuickQuote" : false,
        "destinationCountry" : "UK",
        "departureDate" : "2016-06-08",
        "returnDate" : "2016-06-18",
        "tripDepositDate" : "2016-04-21",
        "rentalCarPickupDate" : "2016-06-09",
        "rentalCarReturnDate" : "2016-06-12",
        "fulfillmentMethod": "email"
    },
    "travelers" : [{
            "firstName" : "Sherlock",
            "lastName" : "Holmes",
            "birthDate" : "1954-01-06",
            "tripCost" : 1740,
            "phoneNumber" : "7152211954",
            "address" : {
                "address1" : "221B Baker Street",
                "address2" : "",
                "stateOrProvince" : "WI",
                "city" : "Stevens Point",
                "country" : "US",
                "postalCode" : "54481"
            },
            "isPrimary" : true,
            "email" : "solving@bhtp.com"
        }
    ],
    "coverages" : [{
            "ratingId" : "FAU",
            "coverageLimit" : 300000
        }, {
            "ratingId" : "CRCC",
        }, {
            "ratingId" : "CFAR",
        }
    ],
}

Coverage Notes

  • CRCC: if CRCC is passed to the coverages array the following fields are required in the policy node:

    • rentalCarPickupDate

    • rentalCarReturnDate

  • FAU: if supplied, the coverageLimit must be supplied in the same object as the ratingId
