XeniApp : API - Improvements

Created by Ronak Jain, last modified on Apr 11, 2022

Below are the few areas where our current Xeniapp API backend can be improved.

  • Use a tempting engine for email templates generation - Right now it’s using string Interpolation method to generate HTML templates which is extremely poor and unmaintainable
  • Move all the Emails sending work to Background queue - Currently, sending emails is a blocking event, we can use background workers to process the emails
  • DB collections have been named haphazardly, without any diligence. To an outsider, it’s often not intuitive. Rename DB collections to make it sound what it does
  • Unit Test coverage - Checking coverage part of the review process
  • APM Integration Sentry / Rollbar
  • Eslint issues - Setup codacy so that eslint issues can be identified during code review
  • Server logging - Winston / Morgan
  • API Request or payload for different client like Mobile/Desktop(Need to be handled)
  • Need to Handle request load for particular API(In Multi cart booking its synchronous that will take much time. it books one by one)
  • API versioning required with proper naming convention ex. api/v1/user etc…
