XeniApp : WordPress plugin for login to booking panel

Created by User cf72c on Aug 03, 2021

Please find the modules listed below,

  1. Plugin structure:

    1. The plugin will be created as per wordpress standards

    2. For this plugin there is no requirement for database extra table interaction at wordpress end

    3. The communication with the booking platform backend will be done with the API calls.

  2. Authentication with booking platform (backend):

    1. We will need the auth token to be generated from booking platform

    2. That token will be entered by the wordpress admin at plugin setting to complete the authentication process

    3. This step is required for security, only the authorized domain should be able to communicate with the booking platform backend.

    4. Here we will need a booking API to verify the token.

  3. Shortcode implementation:

    1. Once authentication process get completed admin will be able to use the shortcode provided by the plugin like [xnp_booking_login]

    2. With this shortcode admin will be able to place the login link anywhere in wordpress setup like in footer, sidebar, pages.

    3. Shortcode will generate the link and popup display.

  4. Login popup design:

    1. The popup will be having login form with two fields: username/email and password

    2. For popup design the colors will be dynamic, based on settings implemented by wordpress admin.

    3. The fonts will be inherited from the wordpress theme.

    4. We will need a generalized design for popup.

  5. Settings at wordpress dashboard:

    1. After the authentication step the admin will get settings for popup color change.

    2. Admin will be able to delete the plugin.

  6. Communication with booking platform for login session:

    1. Once the login popup form gets submitted, wordpress backend will generate the proper call for booking platform backend with encrypted username and password.

    2. This call will be handled at the booking platform backend and necessary login tokens will be generated for verified user details.

    3. On success the user will get redirected to the booking platform admin panel.

    4. On failure, the user will get redirected to login form at the booking platform. There is no need to go back to the wordpress website.
