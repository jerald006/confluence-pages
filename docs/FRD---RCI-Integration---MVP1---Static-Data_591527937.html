Requests logic:

  • Each Super Region code will return the list of properties that offer availabilty

  • For each Super region will be 2 mainly request based on the minUnitSize value (H-hotel; S-Studio)

  • For each type of properties request, it is required to get the complete list of availability offer following:

    • (CurrentDate + 1) + 10 Weeks

    • From first call => (endDate + 1) + 10 weeks

  • Fetch and store the following information:

    • resortId

    • resortName

    • AvailabilityInfo

  • Update Logic

    • It is required to complete the process to get the data and replace 2 times per day (MVP)

    • It is required:

      • 24 regions

      • 6 RQ per region to get the total inventory based on 10 weeks search each

      • Total 144 request to complete the inventory list.

Images content

The quality of the images offer by RCI are not the best. We need to implement a plan b using expedia content but, we need as well to have this option for those properties who don´t match with expedia.

  • Generate a Cache logic to hold the RCI images content associated to:

    • Region

    • Property ID

    • Image category

  • Connect to RCI SFTP to get the list of images per a time frame to refresh the cache.

    • User

    • Password

    • IP Address: It is required to declare the IP address that will connect with the SFTP.
