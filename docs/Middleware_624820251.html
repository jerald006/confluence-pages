Overview

The the objective is to standardize the flight vendors integration implementing a Middleware responsible of translate the XENI Flights API into the vendorÂ´s API calls to get the content and expose in a single way.

This Middleware holds different handles by vendor with RPC to wrapper the content from Vendors data structure to match with the XENI Flights API to hydrate the XENI UI and in a future explode the APIs to different type of consumers.

Solution Proposal

Xeni API

The Flights Xeni API to support the following transactions:

  • flights/airShopping.

  • flights/confirmOption.

  • flights/upsell

  • booking/create

  • booking/update

  • booking/cancel

  • booking/ticketing

  • booking/Queue

  • Post Booking: (InFuture MVP)

    • Shopping Exchange

    • Exchange ticket
