Dashboard

The Dashboard is where the Admin will be able to see a snapshot of the entire platform via Charts and Tables. They will be able to see the following:

Settlements / Day Chart

This chart will contain a history of the amount of settlements per day on the platform. The data on the chart will be data for the current month.

Note: Filters are good to have, but not compulsory

Token Data

This section contains data related to XeniTokens on the platform. It will have the following cards:

  • Total Supply

  • Admin Balance

  • Total Circulation

Recent Bookings

This section contains a history of the most recent bookings on the Platform.

The table will contain the following information:

Booking ID

OTA

Booking Type

Amount

Settlement Status

Token History

This section contains a history of the most recent payments via XeniTokens on the platform.

The table will contain the following fields:

Booking ID

From

To

Amount

Timestamp

Transaction Hash

Best Sellers

Priority: Low

This section contains a table that contains the top users on the platform. Top users are the users who have generated the most amount of revenue.

The table will have a tab at the top which will filter the type of user - OTA, Supplier, Consolidator. The table will contain the following fields:

Name

Revenue

Booking History

When the admin navigates to this area, they will be able to see a history of all the bookings that have been made on the platform. The admin can select any booking from the table and see the entire details of the booking.

A table displays the following information:

Booking ID

OTA

Booking Type

Amount

Settlement Status

Booking Date

Supplier Fees

Aggregator Fees

OTA Fees

Sorting

The admin can sort all the fields by clicking on them to sort in Ascending or Descending order.

Filtering

The results can be filtered using the following fields:

  • OTA Name

  • Booking Type

  • Settlement Status

  • Date Range (Custom)

Searching

Any specific booking can be searched via its Booking ID.

Pagination

Each page will show 20 results and the admin can click on the navigation buttons to view further results.

Booking Details

This page contains the entire details of a booking. When an admin clicks on a Booking Id, they will be navigated to this page. Here, they can see the following:

Hotel/Flight Details

This section contains all the primary details of the booking such as name, dates, etc.

  • OTA Name

  • Name of the Hotel

  • Hotel Address (Line1, City, Country)

  • Hotel Phone No.

  • Booking Date

  • Booking Status

  • Stay Start & End Date

  • Total Fare

  • No. of rooms

  • Room(s) Type

  • Guests (First Name, Last Name, Age)

  • Refundable?

  • Booking Date

Settlement Details

The section following the basic details shows information related to the settlement.

  • Names of parties (OTA, Supplier, Consolidator)

  • Amounts for each party

  • Ledger Details

History

In case of flight bookings or bookings that can have multiple entries, the admin can see a history of transactions.

Payment History

On the payments history section, the admin will be able to view all the payments that have happened on the platform. The payments will be in terms of XeniTokens.

The table for the payments history will contain:

Booking Id

From

To

Amount

Timestamp

Status

Transaction Hash

The status field is used for handling errors/frauds.

Sorting

The admin can sort all the fields by clicking on them to sort in Ascending or Descending order.

Filtering

The results can be filtered using the following fields:

  • Booking ID

  • User

  • Date Time

  • Status

Pagination

Each page will show 20 results and the admin can click on the navigation buttons to view further results.

Invitations

The admin will be able to see all the invitations and their statuses in this area in a table. Since each invitation has an expiry, the admin can resend an invite in case they want to send a reminder.

The table will contain:

Name

Email

Status

Invitation Date

Resend Invite

User List

The admin has the ability to view all users on the platform in a table. The admin can activate/deactivate a user account from this section.

All OTAs get assigned an API Key when they are onboarded. The admin can request for their keys to be re-generated in case of any suspicious activities.

The admin can see:

Name

Email

Wallet Address

Balance

Activate/Deactivate

Regenerate Keys
