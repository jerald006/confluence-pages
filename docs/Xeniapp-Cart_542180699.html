Comments

Current implementation

  • Cart data is maintained at the browser level, all the data is stored in the local storage. When a user logs out, the data from local and session storage is deleted and a new cart is created.

Proposal 1 to maintain/save/load the cart

  • Initial Stage - Empty cart - When the first item is added to the cart, the cart gets added to the database tied to the userid.

  • User login - We show the latest saved cart from the database

  • Cart goes to empty cart state whenever it is saved by the user.

Proposal 2 to maintain/save/load the cart

  • If the user has items in the cart and the user is trying to close the browser or log out, we display a message saying - Do you want to save the cart for future use. If a user saves, then it is stored in the database or else it is discarded and the user gets an empty cart next login

  • User login - We show the latest saved cart from the database

Scenarios

Current Implementation

Proposal

30 mins are done before clicking on book

no checks are done. this will fail in case of car rental.

When user clicks on book button, we check the time between last refreshed and current time, if more than 30 mins, we recheck the availability of each service and inform the user of any price change.

If we aren’t able to find any of the service, we highlight that item from the cart and redirect them to the search page of that service.

Few services gets booked, others don’t

While booking, we are charge the credit card separately for each booking. If any of the services fails, we say booking failed and amount is refunded.

  1. Check for flight change of price - When a user clicks on Book Button - Check whether the flight is still available at the same price or not, if not, display the new price to the user

  2. Book flight first - even if other services fail, the flight can be voided within some time frame. Once the flight is confirmed, book hotel and car rental. If flight fails, not book anything. Inform user to add new flight.

3. If only flight gets booked rest services fails, Show a Void button and ask the user to void the ticket immediately if they wish to. Ask them to contact the support team for help.

Support Team Role

  • Need a system that captures all the failure logs to which the support team has access.

  • If only the flight gets booked - the support team can void the ticket.

  • For multiple hotels or car rental failures - The support team offers alternate options. If the agent/traveler wants to cancel the booked services, then the support team contacts the vendor for the booked services, if the vendor agrees to a full refund, a refund is initiated
