XeniApp : Trip add API details

Created by sonali, last modified on Oct 09, 2023

React component: https://www.npmjs.com/package/@tripadd/components
API documentation: https://api.tripadd.com/docs/#tag/Bundle-Offers/operation/createBundleOffer
Postman Link: https://documenter.getpostman.com/view/716998/2s946o4p2c
Flight confirmation page deisgn: https://www.figma.com/file/xkSoBHUzlo3fywieennr4K/Xeni-Command-Center?node-id=2482%3A19911&mode=dev

  • Create bundle offer:

    • First we need to call create bundle offer API with flight details to get the products from bundle.

    • Flight details contains flight departure, arrival, passenger details

    • This API will give the products array in response

  • Need to send received products array to Card component which we get from tripadd react module.

  • This will load their products in their widget

  • User will select the product and as per products selection booking price will get changed

  • Create order:

    • Once user clicks on booknow and service get booked, then system will call create order API with selected product IDs and passenger and customer(lead passenger) details to create order

    • System will store the orderId in db after booking

  • Get order:

    • We will call get order API using order id to get the order details

  • Cancel order:

    • When user cancel the booking then we need to call cancel order API using order ID

    • Need to check cancellation policy with vendor

  • Please check postman collection for API requests and responses

Markup:

  • We will apply markup considering both (platform and agency markup) on product on TripAdd dashboard

  • After booking we will separately calculate the commission for platform and agency based on the default markup

Questions:

  • We can update products markup on TripAdd dashboard only. We can not update it dynamically. So we need to define default Tripadd markup for platform and agency like refund protect at our side. Because we are getting product prices from vendor and we can not modify it.

    • Platform Markup: 12% (As per discussion with sachin)

    • Agency Markup: 8% (As per discussion with sachin)

  • Also needs to define if we going to apply markup on total bundle price or on every product price of bundle

  • If wants to show products on confirmation page then need design for it
