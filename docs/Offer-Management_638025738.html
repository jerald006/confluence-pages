Summary

The shopping of a trip from Mystify API is based on 2 endpoints:

/api/v2/Search/Flight

/api/v1/Revalidate/Flight

The 1st will return the list of offers that has to be cache, the second is the revalidate of the data, it means that the offers still active or expired for availability or any other changes.

AirShopping

The air shopping is the endpoint that will satisfy the offers from a search params entered from a user in a widget.

/api/v2/Search/Flight - Response

Structure of Response

It is divided in 7 components:

  • FlightSegmentList

  • ItineraryReferenceList

  • FulfillmentDetailsList

  • PenaltiesInfoList

  • PricedItineraries

  • FlightFaresList

  • GroupedItems

Revalidate Offer.

Based on the identification and association of the components for each Itinerary (Journeys), the application will resolve and present the Flight Offer Card.
Each optioned resolved has to be Cache for a period of time and before confirm the offer to the user it is required to revalidate that offer using the endpoint /api/v1/Revalidate/Flight

Params:

  • ConversationID: Unique Shopping Identifier.

  • Target: Production | Test

  • FareSourceCode: Price Token
