A story must follow the INVEST principle

I - Individual : self contained and related to an epic but not another user story

N - Negotiable : should allow room for discussion if not clear

V - Valuable : should deliver value to stakeholders

E - Estimable : must be able to estimate size i.e. story points

S - Small : not be so big as to become impossible to plan/task/prioritize within a level of accuracy

T - Testable : description should provide necessary info to make test development possible

Acceptance Criteria must follow the MoSCoW principle which is represents feature details as:

must have

absolutely necessary

should have

part of the scope but can be worked/implemented at a later date

could have

not part of the scope but can be implemented if circumstances permit

wont have

not part of the scope of services, but can be part of a later implementation.

I created a Story template in Jira called Issue Template which automatically populates the first table and details of the feature must be manually filled along with the Dev and QA tasks

I have listed the details below with Screenshots for creating a story with the template:

  1. Click on ‘Create’

  2. Project defaults to Xeniapp, Issue type should be Story

  3. Add Summary which is name of feature. Do not add anything in the description.

  4. Add other details like attachments, components, linked issues, priority, labels, epic link

  5. Select Create on bottom of the screen

  6. Once the story is created, click on the story link and click the … right hand side top corner of the page and select Apply template

7. Both the templates are the same so select either one of them by clicking ‘Apply’ and close on the bottom.

8. It will take a moment and create an issue with the template i.e. 1st table.

9. Fill the description of all the fields

10. Next you have to create the Dev and QA tasks

11. Click on Add Checklist

12. Creates a checklist to add items

13. To add headers like Dev/QA Checklist, type Dev Checklist in ‘Add ToDo item or header’ and click on the 2nd label next to it and click the tick mark which creates it as a header.

14. Next create the tasks underneath by entering each of the tasks.

15. For QA checklist, follow Step 12 followed by 13.
