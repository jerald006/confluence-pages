Functional objective

Offer to the consumer to select and pre-reserve the seats for their flight segments booked.

Prerequisites

  • Reservation Created

    • Itinerary segments

    • Fare Segments

    • List of passengers

    • Passenger Types

    • Fared booked (allow pre-reserved seats)

Functional flow

Actor

Action

Authenticated User

Select an Itinerary

Book the itinerary

Complete the passenger’s information

Identify each passenger with the proper PTC (Passenger Type code).

Below Passenger Information - Create a section “Select seats”

Passengers expand the section and see the Seats component by flight Segment and a Dropdown list with each passenger’s Name and PTC

In the passenger’s Dropdown, grey-out - PTC = INF (Infant not occupying a seat)

Identify available seats and greyed-out occupied seats.|

Mouse over a seat, and display a description tip, with the characteristic and price of the seat.

The user has to select each seat for each passenger on the list.

If the itinerary has more than one segment:

  • Create tabs by segments.

  • Each segment should resolve the seat map.

  • The passenger dropdown has to be the same list of passengers.

  • The passenger should select only 1 seat passenger per segment.

  • Each seat selected with the associated price must appear on the trip summary and update the total price of the trip.

The user continues with the Booking flow
