XeniApp : API Authentication

Created by User d8ec2 on Jun 09, 2021

Location

POST /v1/auth/login/{AgentCode here}

Headers:

Content-Type: application/json

Request 

{
    "username" : "{username here}",
    "password" : "{password here}"
}

Response

200 OK

{
    "userName": "...",
    "token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJBZ2VudENv...NzM3OH0.5Fb49hI-OxcVQpdIfG03xzBrS6k0Yl61fTHp4okVgH8",
    "agentCode": "...",
    "agencyCode": "...",
    "message": "Authentication Approved"
}

Once you have received a 200/OK response, you will receive a token in the response body.

This token is to be used in the "Authorization" header of every subsequent request using the Bearer schema. Authorization tokens are good for 24 hours so make sure to re-authorize and cache each day for best performance.

Example Authorization Header

Authorization: Bearer eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJBZ2VudENv...NzM3OH0.5Fb49hI-OxcVQpdIfG03xzBrS6k0Yl61fTHp4okVgH8
