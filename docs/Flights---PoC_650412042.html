XeniApp : Flights - PoC

Created by Gerardo Nizetich, last modified on May 20, 2024

Objective

The proof of the concept for the flights revamps integrating MYSTIFLY as the vendor.
The idea is to generate a model to integrate future vendors into a unique UI design.

What we need

  • Create the Flights API based on the Standard Model of the Industry to exchange flights and pricing information. (Possible Model: )

  • Create the Flights Dashboard for super admin flight-vendor management. (Market Tables)

    • Sections to manage flights content:

      • Vendor Name

      • OfficeID

      • Currency of operation

      • Enabled Markets

        • Enable Routes per market

      • Applicable sales conditions.

      • Prefered carriers.

      • Restricted Carriers

      • Rate vendor. (Assign a ranking to each vendor for future use)

  • Create a Middleware responsible of interpretare the business rules (marketing Tables) and get the content from the supplier.

  • Create Cache and update cache data logic.

  • Adjust the UI design based on the data. (Using Emirates as model)

    • Choosing departure flights

    • Select return flight filtered by the Selected departure flights.

  • Add the seat map section, Ancillaries section in the booking flow.

  • Create the split payment logic when it is applicable.

  • Create the metrics for consumer (Analytics) to measure the funnel of the Flights flow.

  • Create the internal metrics to measure the Vendors consumption and content by Market, Carrier, Sales, Segments, Search vs Book, Ticketing Vs Cancellations, Payment Errors, Handling errors).

  • Create the flights activity logs for error controls.

    • Non-Transactional: Searches (Low period of time)

    • Transactional (More period of time)
