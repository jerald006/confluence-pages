XeniApp : Issue List

Created by Praveen Shivalingaiah on Nov 02, 2022

Open Items and Issues identified in Xeni and Xenipay ecosystem

 

  1. Idempotency issue in booking

  2. Atomicity issue in booking

  3. Frontend controlling the booking amount, no validation of the same in backend

  4. Business logic is segregated in frontend and backend

  5. Issue in Error handling resulting in loss of business for Xeni

  6. Broken bridge between Xeni and Xenipay (Retries not present)

  7. Code is too tightly coupled to introduce new concepts (example: booking issue with bitpay)

  8. Code cleanup required in all systems

  9. E2E logging, monitoring and APM - Sentry is being used here.

  10. Devops automation in Xenipay

  11. Proper unit testing / code coverage

  12. API rate-limiting issue

  13. graceful shutdown

  14. System Recovery operations

  15. Need to make this an event-driven microservice gradually.

  16. Consolidator onboarding vs getting list from Xenipay

  17. SSO for the entire system

  18. Customer mapping on Xenipay - Currently the customers that are created in Xenipay are not properly mapped to the OTA they are created on, as a result, technically an OTA can access details of any customer.

  19. Internationalisation challenge - Supporting Internationalisation on the current platform will be a big task

  20. Globalisation challenge - Supporting LTR and RTL UI will be a big task

  21. Different UI for mobile and desktop. Need to identify the reason for the same and work on making it liquid responsive.

  22. Caching is missing in the system which will create load on the system with increasing number of concurrent sessions

  23. Cancel booking integration in Xenipay is pending

  24. Bitpay refund? How to handle?

  25. Is Sales Agent onboarding required in Xenipay? What is their role in Xenipay?

  26. Auto-settle notification to Xeniapp from Xenipay is pending.

  27. Documentation

    1. Business Documentation

    2. Technical Documentation

  28. Should we extract Payment Module as a separate service that can work independent of Xeniapp?
