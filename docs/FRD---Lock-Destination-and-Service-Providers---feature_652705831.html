    • How Agency owners lock destinations:

    It is required to create a section in the Agency Admin settings to allow Agency Owners to set the Lock feature, also, they can add more than one configuration instance to setup different Lock destinations as follow:

    • Name: User has to give a name to the Lock Destination instance.

      • Country: The user can lock the entire country or select what to locks for the selected country:

        • Regions: The user can create a Lock region that could include a single or more States/provinces and also by City of each selected item.
          A region can have cities, states/provinces(more than one), and both.

          • State/Province: The user can lock an entire state/province or certain cities individual. It is required to pre-populate the list of Cities in a State/province.

            • City: The user can enter a single or a list of cities from a State/Province. It is required to pre-populate the list of cities based on the selected state/province.

    Lock destination setup

    When an Agency Owner has setup the lock destinations, in the Application all the From/To combos have to hide the values previously selected in the configuration.

    The configuration should be stored in the DB of the tenant focus on each individual destination selected and assign the configuration an ID.
    The user has to have the capability to keep more than one configuration of lock destinations with Different Ids.

    Agency Admin Lock Destinations

    When the agency owner, has already setup lock destination and return to the section, it has to show the configuration instance and could edit and also delete the entire configuration ( Soft-delete) under a single Name

    Hotel Service providers

    As part of the effort of the Lock feature, it is required to allow only for Hotels service, the capability for Agency Owner /Agency administrator to exclude some particular hotel chain or properties.

    This part of the feature implicit a Filter over the content received from the Vendor based on the configuration entered in the Tenant.
