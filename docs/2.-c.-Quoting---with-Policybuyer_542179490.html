XeniApp : 2. c. Quoting - with Policybuyer

Created by User d8ec2, last modified on Jun 09, 2021

The policybuyer node is used to provide information about the person buying the policy. It allows for some ownership of the policy when: the buyer is not the primary insured traveler (might be a traveling companion of the primary insured traveler) or when not traveling and purchasing for someone else.

If the policybuyer node is not supplied, it is assumed that the primary insured traveler (the traveler with isPrimary set to true) is the policybuyer.

Request

{
    "policy" : {
        "productId" : "EC",
        "isQuickQuote" : false,
        "destinationCountry" : "UK",
        "departureDate" : "2016-06-08",
        "returnDate" : "2016-06-18",
        "tripDepositDate" : "2016-04-21",
        "rentalCarPickupDate" : "2016-06-09",
        "rentalCarReturnDate" : "2016-06-12",
        "fulfillmentMethod": "email"
    },
    "policybuyer": {
        "firstName": "Irene",
        "lastName": "Adler",
        "birthDate" : "1950-08-27",
        "email" : "clues@bhtp.com"
    },
    "travelers" : [{
            "firstName" : "Sherlock",
            "lastName" : "Holmes",
            "birthDate" : "1954-01-06",
            "tripCost" : 1740,
            "phoneNumber" : "7152211954",
            "address" : {
                "address1" : "221B Baker Street",
                "address2" : "",
                "stateOrProvince" : "WI",
                "city" : "Stevens Point",
                "country" : "US",
                "postalCode" : "54481"
            },
            "isPrimary" : true,
            "email" : "solving@bhtp.com"
        }
    ]
}

Notes

  • We recommend policybuyer's birthdate to reduce false positives on OFAC, but it's not required.

  • Policybuyer's email is not required but is suggested for the management of the policy in the future.
