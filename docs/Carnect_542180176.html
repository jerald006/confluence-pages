List of API endpoints that we hit from FE:

CAR_RENTAL_COUNTRIES: '/api/car-rental/findCountries/EN',
	-to display the countries info
CAR_RENTAL_STATIC_DATA: '/api/car-rental/getStaticData',
	-to display all the static content in all car rental
CAR_RENTAL_SEARCH: '/api/car-rental/availableCars',
	-to display car info on the landing page
CAR_RENTAL_EXTRAS: '/api/car-rental/carDetails',
	-to display car info on extras page
CAR_RENTAL_BOOK: '/api/car-rental/reserveCar',
	-to reserve a car
CAR_RENTAL_MYTRIPS: '/api/car-rental/findMyReservations',
	-is for the dashboard
CAR_RENTAL_FIND_RESERVATION: '/api/car-rental/findReservation',
	-is to display the view details page
CAR_RENTAL_CANCEL_BOOKING: '/api/car-rental/cancel',
	-is to cancel the car booking

Whitelist of IPs:

In order to run the api locally, Carnect needs to whitelist the IP address. So, we used the Car Rental sandbox on AWS for developing the api.

Testing guidelines:

Below are some of the testing guidelines to follow:

  • When you try to book a car rental for testing, make sure to select a date that is six months later to the current date.

  • Make sure to cancel the booking once the validations are done.

  • The age selected in the home page should match with the Date Of Birth entered in the multiplebooking page.

  • Prices vary with the Country Of Residence that is selected.
