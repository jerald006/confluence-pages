XeniApp : Custom package flow

Created by sonali, last modified by Ayush Kumar on Oct 09, 2023

Design link: https://www.figma.com/proto/xkSoBHUzlo3fywieennr4K/Xeni-Command-Center?node-id=345-5541&scaling=scale-down-width&page-id=0%3A1&starting-point-node-id=37%3A2181

General notes:

  • All packages will be considered as group packages

  • Agent will be able to see only the packages created by self.

  • Other agent's packages will not be visible on other domain

  • Age will not be considered in packages booking, pricing are same for all participants

Package listing for Agent:

  • On command center there will be menu in sidebar navigation

  • On package listing page all packages created by current logged in agent will be displayed

  • Agent will be displayed based on creation date, recent to old with pagination

  • Package statuses: Draft, Published, Deleted (Deleted packages will not be visible)

  • Once published and has booking then it can not be return to draft status.

  • Packages will be editable only if there is no booking against it.

  • Delete package will be irreversible process, so while deleting package there will be double confirmation message.

  • Agent will be able to share package link on social media

  • Single package will be having stand alone link which is accessible without login. It will represent user view of single package.

Package creation:

  • Title:

    • Required

    • Max: 30 Char

    • No special characters

  • Tagline

    • Optional

    • Max: 120 Char

  • Description:

    • Required

    • Max: 120 Char

  • Tags

    • At least one tag should be selected

    • Max 8 tags will be attached to single package

    • This will get selected from pre-defined tags list

  • Start location

    • Required

    • Google integration for auto select

    • Need lat/long in database

  • End location

    • Required

    • Google integration for auto select

    • Need lat/long in database

  • Duration

    • Required

    • Max: 15 days

    • Based on duration itinerary days will be populated 

  • Departure dates

    • At least one future date is required to publish the package

    • Max: 25 dates (Future available dates)

    • Calander display for selecting date

    • Display all added dates at create, edit package

  • Retail price

    • Required

    • Only numbers

  • Wholesale price

    • Required

    • Only numbers

  • Agency commission

    • Required

    • Only numbers

  • Price to Traveller / Person

    • Required

    • Only numbers

  • Capacity

    • Required

    • Only integer

    • Based on this number of seats left will be calculated

  • What’s Included

    • Optional

    • Max number of fields: 10

    • Character limit for each field: 120

  • What’s excluded

    • Optional

    • Max number of fields: 10

    • Character limit for each field: 120

  • Package Brochure or Documents

    • Optional

    • One file per field, max three fields

    • Allowed file formats: Doc, docx, pdf

    • Max size per file: 8 MB

  • Video Link

    • Optional

    • Full url of website, check format

    • Single url

  • Package images

    • Optional

    • Max items : 12

    • First image will be considered as featured image (To be shown on list views)

    • Max size per image: 2 MB

  • Age Restriction

    • Optional

    • Integer only

  • Cancellation Terms

    • Optional

    • Max : 120 char

  • Other Terms

    • Optional

    • Max : 120 char

  • Itinerary

    • The number of days will get generated based on user input for duration

    • Please refer the wireframe for field and values for event creation popup:
      https://www.figma.com/file/hCJ6uTNjLDhXrVY8e5n4uw/Xenivoyage-package-and-Booking-flow?node-id=0%3A1&t=ljpTiCG3oM5LxigV-0

    • Please follow general rules for text and numeric fields as above.

    • Orders of itinerary items can be change in single day or to different day section

Agent package listing

  • On command center all packages created by agent will be displayed

  • Basic information of each package will be displayed in table format like: ID, title, image, creation date, status, price, location, action

  • In action there will be, view preview, share package publicly accessible link on social media, edit package, delete package, duplicate package.

  • In bulk action, package delete will be available.

Customize for me option:

  • This will open form with following fields:

    • Name

    • Email

    • Phone number

    • Message

    • Package reference id/link (hidden)

  • This will get submitted to the respective agent as well as to the client for successful submission

  • Further conversation we consider it off the system

Package booking:

  • Package search:

    • There is a tab for packages besides activity on book now

    • Users will enter the location and when clicks on search packages will get displayed based on the location

    • Filters:

      • Price range

      • Duration

      • With or without flight

      • Interest (Tags that attached to packages while creating package)

    • Once the user clicks on any package it will redirect to the details page

  • Package Details:

    • This page will have all package info in detail which contains:

      • Images

      • Description

      • Included, Excluded

      • Day wise Itinerary Info

      • User can download brochures which uploaded while creating package

      • Future available dates

    • On this page, the user will select the date, add passengers and check the availability of the package. User can directly choose dates from future available dates also which are shown on the page

    • Once clicks on Book Now system will redirect to the checkout page

    • User can also book packages from the single package link which is shared by agent. This is public link. So once the user clicks on Book Now first system will ask to log in and after logged in system will redirect to checkout page

  • Checkout:

    • User/ Agent is able to change the dates and passengers on the checkout page also

    • If logged in as an agent then he can add custom markup on the checkout page

    • User/agent will fill passenger details and billing details and book the package
