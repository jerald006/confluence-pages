Context

To improve performance, cache the static data per each ship of the enabled cruise line for Xeni.

Authentication

The first step to consume the FusionAPI of TravelTek vendor is to get the accessToken based on Xeni credentials: Cruises Integration (API Details)

Request example

curl --location 'https://fusionapi.traveltek.net/2.1/json/token.pl' \
--header 'Content-Type: application/x-www-form-urlencoded' \
--header 'Accept: application/json; charset=utf-8' \
--header 'Authorization: Basic eGVuaTp4czgxZHM1NA==' \
--data-urlencode 'scope=portal' \
--data-urlencode 'grant_type=client_credentials'

Response Example

{
    "expires_in": "7200",
    "access_token": "97218e52p17f8-4e4c-b05f-c0c3a368bac9",
    "token_type": "bearer"
}

access_token: It is required to pass to the session payloads as the value of the param requestid

expired_at: The time of access_token live. Expressed in Seconds. Based on this it is required to create a logic of access_token refresh and control the user session as well.

Cache Job.

Based on the static data of each Cruise line enable for Xeni, it contains each shipId by Cruise line.
To get the static data, it is required to iterate the /cruiseshipdetails.pl method.
