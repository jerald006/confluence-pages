XeniApp : Viator APIs Integration Doc

Created by User c6d9e on Jun 22, 2022

Goals

The goal of this document is to give overview of XeniApp activities wrapper APIs based Viator merchant APIs.

Flow diagram

API Details

Destination search

  • This api returns list of Viator destinations including names and parent identifier.

  • Core is running in background which runs DAILY(recommended frequency) to fetch new available destination and store it in our database.

Product Search

  • Based on selected destination list of product will be searched.

  • Here we can filter products by Price high to low, low to high, Flags the complete list of filter can be found here

  • For products using the ingestion model for connecting to Viator’s supply. In this model, we are maintaining a local database of products and also implemented cronjobs which calls to Viator to ensure that destination, products and availability information is up to date.

Product Details

  • This api get full product details for all products modified since a specified time.

Availability Check

  • This api is used for checking product availability for travel date for selected number of passengers full list of params can be found here.

Check product Schedule

  • This api is used to check product schedule for upcoming days with pricing options

Booking

  • This api is used for booking product

Cancel Booking

  • This api is used to cancel booking

Booking Status

  • Cronjob is running every two minutes to update booking status of product as part of ingestion model.

  • This corn will update both booking and cancellation status
