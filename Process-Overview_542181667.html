Booking

Create Booking

When XeniPay receives a booking from an OTA, it does the following:

  • Verifies if the payment has been made for that on ConnexPay

  • Stores Booking Details on Blockchain

Settle Booking

After a booking has been created, it can be settled automatically, or by the OTA that created the booking.

A booking is auto settled if it is non-refundable or if the booking date has been reached.

To settle the booking on demand, the OTA can ask XeniPay to settle the booking. When a booking is settled, XeniPay does the following:

  • Stores a settlement event for the booking on Blockchain

  • Transfers Xennies to each party specified during booking creation

Redeem Booking

Any stakeholder can convert their xennies from settled booking to Fiat currency. Any booking that has been settled can be redeemed.

Note that a stakeholder can redeem multiple bookings at once. But, the total amount should be more than $100.

When a booking is redeemed, XeniPay does the following:

  • Stores a redeem event for the booking on Blockchain for the stakeholder

  • Transfers Xennies back to the XeniPay treasury

  • Transfers Fiat to the stakeholder via ConnexPay ACH
