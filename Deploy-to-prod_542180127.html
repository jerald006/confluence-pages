Production Deployment Steps

  1. In xeniapp-infra-backend-prod project update the following variables replacing the domain name

    1. top_level_domain_name

    2. company_url

    3. forgot_link

    4. base_url

  2. From xeniapp-infra-backend github project create the latest prod release

  3. Review terraform plan in Terraform cloud and approve.

  4. From xeniapp-api github project create the latest prod release

  5. Test if server same up by checking health endpoint

  6. Login to the box backend-server-prod-0 and follow the database migration steps.

  7. After destroying the database and before running the migration scripts confirm in mongo db atlas that the user app-user-prod has read-write access to biirdee-db-prod. Note : the script “40_update_stripe_customer.js” takes a long time

  8. After DB migration spot check agencies , user collections were updated correctly

  9. In xeniapp-infra-frontend-prod project update the variable “top_level_domain_name” replacing the domain name

  10. From xeniapp-infra-frontend github project create the latest prod release

  11. In github update the organization secret “PROD_TOP_LEVEL_DOMAIN” to the new domain name

  12. From xeniapp-react github project create the latest prod release

  13. From xeniapp-admin-dashboard github project create the latest prod release

  14. After deployments perform checkouts for command-center and super-admin

Deploy Code (Follow the same for all 3 code-repos)

Make sure latest infra repos have been deployed esp. xeniapp-infra-backend xeniapp-infra-frontend.

  • Fetch last deployed tag into QA (last 3 in example below sometimes same sha is deployed multiple times with different tags)

➜  xeniapp-api git:(agency-subscription-flow) git fetch --all && git --no-pager tag --sort=-committerdate --list 'qa-*' | head -3
Fetching origin
qa-0.0.25
qa-0.0.24
qa-0.0.23

  • Find commit hash of the deployed version

➜  xeniapp-api git:(agency-subscription-flow) git show-ref qa-0.0.25
cf8ae165c840bc435ad90e2c1fa3f5a2a2b1465a refs/tags/qa-0.0.25

  • Find last tag deployed to Prod (last 3 in this example)

➜  xeniapp-api git:(agency-subscription-flow) git fetch --all && git --no-pager tag --sort=-committerdate --list 'prod-*' | head -3
Fetching origin
prod-0.0.6
prod-0.0.5
prod-0.0.4

  • Create new release (prerequisite: Github cli should be installed ). Use the commit sha received from the show-ref command and increment the Prod tag. Read comments # on the individual steps

➜  xeniapp-api git:(agency-subscription-flow) gh release create prod-0.0.7 -t prod-0.0.7 --target cf8ae165c840bc435ad90e2c1fa3f5a2a2b1465a
? Title (optional) prod-0.0.7 # press enter to select default
? Release notes Write using commit log as template # select this option, press ctrl+X to close nano editor
? Is this a prerelease? No # select no
? Submit? Publish release # publish release
