Pro & cons

Pro:

  • Good API Structure

  • Easy to integrate based on that structure

  • Complete transactional services (Shopping flow, Booking Flow, Post-booking, Webhooks, etc).

  • Flexible search params

  • On hold bookings

  • Get ancillaries list for selected Offer (Baggage, Meals, Speedy_boarding, Checkin-charge)

  • Support Seat map (Chargeable seats is not allowed to book, only GDS content) - In vendor development.

  • Standalone order Ticket API

  • Cancel Booking Service

  • Void service

  • Reissue quote

  • Multi-City separate API

  • Schedule Change handling API

  • Errors detailed

Cons

  • Non-flexibility with Currencies (Do not support a property to rule the currency in the request)

  • Create a POS - SITE per Currency

  • Lack of performance (Revalidate offer)

  • No chargeable seats supported.

  • Unknown available FOPs (Form of payments)

  • Consolidator services works with Deposit only.
