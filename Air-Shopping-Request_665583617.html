XeniApp : Air Shopping Request

Created by Gerardo Nizetich on Jun 05, 2024

The endpoint to process the air shopping call for the vendor is:
/v2/Flights/Search

Please note that the endpoint to process multi-destinations is different.

Request by Cabin:

Each request is limited to the service Cabin class. The vendor supports in the response only 1 cabin per call, which means to resolve each search it is required to execute 4 calls (Economy, Super Economy, Business, First) to get the total content data

Under the Tag TravelPreferences is must be declare like:

"CabinPreference": "Y", => Cabin to get the content preference
        "Preferences": { => How the preference is applied
            "CabinClassPreference": {
                "CabinType": "Y",
                "PreferenceLevel": "Restricted" => Get only the content of the CabinType declared.
            }
        }

Currency

Mystifly not support the multi-currency. The site for XENI is a single site configured in USD
The exchange currency to display according to the XENI Site preference of the end-user is charge to XENI effort.
