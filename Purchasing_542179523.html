XeniApp : Purchasing

Created by User d8ec2 on Jun 09, 2021

Purchasing runs through the same processes as quoting does to ensure the quote is valid to be purchased. However, purchase requests also require payment information.

Coverages

When purchasing, you must provide all optional coverages within the coverages array that are intended to be offered with the policy. Only the "ratingId" and the "limit" are required (limit is not required if there is only one available limit).

Payment Information

the "paymentInformation" node is where all information about the purchase is provided.

Sample Request & Response

Location

POST /v1/purchase

Headers

Content-Type: application/json
Authorization: Bearer {your-token.from-the-authentication.response-here}

Response

Upon a successful purchase call, a policyNumber will be returned. The response will look exactly the same as the quote response but there will be a non-zero policy number.

200 OK

{
    "policyNumber" : 200001518,
    ...,
    "coverages" : {
        "Included" : [...],
        "Optional" : [...]
        ]
    }
}
