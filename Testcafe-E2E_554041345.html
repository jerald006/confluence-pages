Project Structure and Writing Tests:

We are following Page object Modal framework.

src folder has separate folders for desktops and mobiles. Each folder has pages, tests and utility folders.

Each module has page file in pages, which looks like below:

Page File: 

export class SamplePage {
  constructor() {
   this.selectorName = Selector("cssSelector");
   }
   
   async function() {
    await t.click(selectorName);
   }

Each module has test file in tests, which looks like below:

Test File: 

import { SamplePage } from '../pages/sampleFile';

const samplePage = new SamplePage();

fixture 'Fixture description'
 .page(url)
 .beforeEach()
 .afterEach();

test.meta('suite', 'Suite1')('Test case description', async () => {
  // actions to be performed
  SamplePage.function();
});

Execution of Tests:

Steps to Run Tests From the Local Machine:

  1. Clone the QA automation repository

  2. Node.js and Package Manager (npm) : Install Node.js from their official website Or Install Node: js using command line. Go to the terminal or command prompt & run the below command.

  3. Run the below command in the terminal to install the node_modules

     npm install

  4. After installing, you can run the scripts with the test suite shortcuts provided in the package.json. For example,

     npm run BrowserStack:WinChrome

Steps to Run Tests from GitHub Actions:

  1. Navigate to Actions tab in GitHub

  2. Select the workflow as per the OS and browser execution

  3. You can manually Trigger the execution

  4. It will create Test Execution reports in the Xeni 3.0 Jira project

Test Execution Reports:

You can find the Test Execution reports here named in this format : Test Execution for <OSType BrowserType Today's Date>

For example, Test Execution for Windows Chrome Feb-24-2023
