1.   User Stories

1.1. Cart 

1.1.1.     Cart Details and number of Products

As a user I want to add multiple products in the cart so that I can book all products at once.

Acceptance Criteria

Each time user clicks ‘Add to cart ‘button

  • Products gets added to the cart and respective details are added

    • Flights- From/To, Departure/Arrival date & time (12 hr format), Airline name, Baggage, Number of travellers, total flight duration, Number of stops (hyperlink to display- Stop details), Class (info icon for Cabin details), Price for each flight, Flight total

    • Hotels- Hotel name, Hotel address, Star rating, Check-in/Check-out Date, number of days stay, Room quantity and type, number of guests, Non-refundable(info icon to display cancellation policy) , Hotel total (info icon to display Hotel price details including taxes and fees)

    • Car rental- Vehicle Name & type, Agency name, Pick-up/Drop-off location, date, and time (12 hr format), Number of passengers, baggage and doors, cancellation policy (info icon to display cancellation policy) , Car Features (AC, Automatic), Car total (info icon to display car rental price details including taxes and fees)

  • Max 12 Products allowed in the cart for any combination of products NOTE: More product types will be added but the total number of products will be 12 max(any combination) (Sarika to verify again the max limit of 12 products in prod after the fix for car stripe error is deployed)

    • Flights: Max number of flights is 4. Any number of Pax combination must not exceed 9

    • Hotel Rooms: Max 4 Hotel rooms allowed to be booked

    • Car Rentals: Max 4 car rentals allowed to be booked

    • Display pop-up message when limit exceeds

      • Users adds more than 9 pax collectively in all flights - Display message “Max allowed passenger limit is 9”

      • Users add more than 12 products (any combination) in the cart - Display message “Max allowed product limit is 12”

      • Users add more than 4 flights in the cart - Display message “Max allowed flight limit is 4”

      • Users add more than 4 hotel rooms in the cart --Display message “Max allowed hotel rooms limit is 4”

      • Users add more than 4 car rentals in the cart - Display message “Max allowed car rental limit is 4”

1.1.2.     Total for each product and overall total price

As a user I want to view price for each product and total price so that I can estimate total costs in the cart before the checkout

Acceptance Criteria

  • In the cart, the user can view

    • Total with currency for each product types including all taxes/fees

    • Total with currency for all products including all taxes/fees

  • Update the overall total price if user add/removes any product-TBD new functionality of adding markup on overall total price

1.1.3.     Remove Products

As a user I want to remove individual products from the cart so that I can book only required products

Acceptance Criteria

  • Displays the Trash button to remove individual product from the cart

    • Display confirmation message

    • Update overall total for the product type and the grand total including all taxes/fees

1.1.4.     Clear all products

As a user I want to clear the entire cart so that all the products are removed at once and the cart is blank

Acceptance Criteria

Displays Clear Cart CTA to remove all products

  • Display confirmation message

  • Display blank cart

1.1.5.     Persist cart

As an agent/traveler, I like to restore my products in the cart when I close and re-open the browser so I can check out with existing products in the cart

Acceptance Criteria

  • When user closes the browser and opens the browser again

    • Restore the latest cart

    • Check for availability and price change (Refer to Checkout button user Story- X3UI-99 - Getting issue details... STATUS )

    • Reset the Cart Expiry Time to 20 minutes (default)

  • When user logs out and re-login

    • Display blank cart

1.1.7.     Checkout Button

As a user I want to click the checkout button so that all the products in the cart are re-checked for availability and price changes before final booking

Acceptance Criteria

  • When there is change in availability

    • Display message to notify the user “The product (specify flight arrival/departure date and time /hotel name check-in/checkout date/car brand and type pickup/drop-off date and time) is no longer available, please remove the product and find another option”

      • When user clicks “OK”, stop the checkout process

  • When there is no change in availability

    • Continue checkout process and display checkout page

  • When there is change in Price

    • Display message to notify the user to accept price change or remove the product

      • Accept price change- Update the total price and overall total price in the cart. Continue checkout process

      • Remove Product: Remove the product from the cart and continue checkout process

  • When there is no change in price

    • Continue checkout process and display checkout page

1.2. Checkout Page 

1.2.1.     Cart Expiry timer

As an agent/traveler, I want to book the products within the expiry time limit and could refresh the timer so that the products can be booked successfully

Acceptance Criteria

  • When user clicks on checkout button, system must start the timer on checkout page

  1. Default timer expiry is set to 20 minutes

  2. When timer expires, display the message “Your cart has expired click refresh to re-check availability/pricing”. NOTE: User cannot exit this pop-up without clicking Refresh button i.e there should not be a cancel button for user to exit without refreshing the cart.

A. When user clicks on ‘Refresh’ button

i. System must check the availability and price for each product in the cart (Refer to Checkout button user Story- X3UI-99 - Getting issue details... STATUS )

ii. Reset the timer to 20 min

  • When the user clicks on ‘Refresh Cart’ button then the system must re-check the availability & price and reset the timer to 20 min

1.2.2.     Products Summary

As a user I want to review the products in the cart so that I can complete booking.

Acceptance Criteria

  • On clicking Checkout button, the user will be redirected to checkout page

  • Display each product details-

    • Flight- From/To, Departure/Arrival Date and time (in 24hrs format), Flight number, Hero image of Airlines, Number of stops, Layover Time, Class and Baggage

    • Hotel- Hero image, Name and address of the property, star rating, Check-in/Checkout Date and Time, Room Type, Special Instructions CTA, Reservation & cancellation policy CTA, Cost per night (calculated based on Total price/no.of days stay), Taxes/Fees , Total price (including taxes and fee)

    • Car rental- Hero image of the car, brand and car model, car type (economy, compact), Price per day, Taxes and Fees, Total Price, Number of passengers, number of bags, door count, Pickup and Drop-off Date, time and location, Rental agency and car features (air-conditioning, automatic) Fuel policy, included mileage and estimated deposit, Collision damage waiver, Theft protection, Rules and Restrictions CTA, Reservation and cancellation policy CTA, Product information CTA

1.2.5.     Pay securely button

As a user I want to click the pay securely button so that all the products in the cart are re-checked for availability and price changes before final booking

Acceptance Criteria

  • To enable Pay securely button, user must agree to terms and condition

  • System must check if the user has entered same passenger first name, last name and DOB for more than one flight with same airline on same date then display a warning message “Same passenger not allowed to book same airlines for same date” and system should not proceed the booking unless passenger updates or remove conflicting flight details - Check for condition where duplicate passengers found with same itinerary

  • Check mandatory fields and payment details are entered on the form. Mandatory fields for each product are:

    • Flight-Title, First Name, Middle Name, Last Name, DOB, Gender, Phone, Email

    • Hotel: Title, First Name, Last name for each guest

    • Car Rental: Title, First Name, Last name, DOB, Gender, PHONE, Email, Address, Country, City, Region, Zip Code, Flight Number, Flight Code

    • Payment Details: Name, Credit card number, Expiry date, CVV

  • System must process the booking for products in the sequence order of: Flights, refundable hotels, refundable cars, non-refundable hotels/ cars (all refundable products must process before non-refundable)

    • Process one product at a time

    • Check Availability and Price changes (Refer to Checkout button user Story- X3UI-99 - Getting issue details... STATUS )

    • If there is no change in availability and price

      • Payment Process: process payment for each product

        • Success: Continue booking process

        • Failure:

          • If payment fails on first product, Display message “Payment failed. Please check payment details.” Abandon payment process and cart remains the same

          • If one or more product is booked successfully, Display confirmation page with details of the booked products and display the failed payment error message for the failed product

      • Booking Process:

        • Success: Continue booking next product

        • Failure:

          • When booking fails,

            • Display message “The product booking failed. Do you want to continue?”. If yes, continue booking process for other products. If no, show confirmation page with successful bookings (if any).

            • Refund payment for unsuccessful booking.

      • Booking confirmation:

        • If one or more product is booked successfully but the booking fails for any of the subsequent products, Display confirmation page with details of the booked products and display the failed booking error messages. Send confirmation email to the user.

1.2.7.     Email Cart- Agent only (***Needs to be updated. Pending discussion for new workflow***)

As an agent I want to email the products in the cart so that the client receives an email along with markup

Acceptance Criteria

Displays Email Cart CTA to link cart to traveller’s profile

  • Display pop-up message to enter email and markup the price in % or $ amount

  • Store emailed carts in Agent command center

New Email Workflow:

Step 0: system to check email id for client

Step 1: products added to the cart by client

Step 2: agent adds markup(percent or dollar amount). add email address

Step 3: click ‘Send proposal’ button . Check for availability and price

Step 4: Client receives the email with itinerary . Client can either approve or reply. On agent side, they get details in CC--> PLUM

step 4.1: Approve- Client re-directed to credit card authorization form. Client confirms the itinerary and enter booking details (including mandatory fields). All the information about the travel/ passenger must be verified twice.

I , authorize (travel agency name) to charge this credit card --- . The actual charge amount may vary +/- 10%

On Agent side : when traveller clicks approve button, agents receive a notification in Dashboard and on clicking the notification , checkout page is displayed with pre-filled information.

Step 4.2: Reply - Display a pop-up message with text box for clients comment. When client enters the comments and hit send button then travel agency receives this client message ( along with unique itinerary number) and then the travel agency will respond to this message

NOTE: If price is within +/- 10% then agent proceeds with payment . If more than 10% change then agent needs to get approval from client again.
